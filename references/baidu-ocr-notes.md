# 百度 OCR 参考笔记

## 主路径
- 扫描 PDF 的主路径应优先使用 **文档解析 / 文档格式转换**。
- 页级图片 OCR 只作为兜底方案。

## 图片 OCR
- 适合单页截图、页图识别、局部回退。
- 通常会先渲染 PDF 页面，再调用通用/高精度 OCR 接口。

## 文档解析 / 文档格式转换
- 适合整份多页 PDF/OFD 的版面分析与 Markdown 输出。
- 支持整份文档提交后异步轮询结果。
- 输出通常包含 `markdown_url`、`parse_result_url`、`task_error` 等字段。

## 关键限制
- API Key / Secret Key 必须配置。
- 需要管理 access token，通常 30 天有效。
- PDF / 请求体有大小限制，超过限制要分批或改用 file_url。
- 免费额度和 QPS 可能较低，批量任务要做重试和限流。

## 建议策略
1. 先检测 PDF 是否已经包含可抽取文本。
2. 扫描件优先做文档解析。
3. 文档解析失败时再做按页 OCR。
4. OCR 或文档解析后输出 Markdown，再做研究/摘要。

## 常见错误码
- 401 / token 无效：重新取 token。
- 429 / QPS 超限：退避重试。
- 5xx / 服务异常：重试，必要时降级。
- 文件大小异常：分割 PDF、改用 file_url，或转按页图片。
